# Nowcast Repository

## Overview
The `NowcastRepository` handles the persistent storage of all scientifically validated events generated by the Nowcasting Engine. While the `MasterCatalog` operates largely in-memory for low-latency real-time state management, the Repository ensures that the data is safely persisted to disk, enabling historical search, replay, and statistical analysis.

## Database Architecture
The repository is backed by a lightweight, embedded SQLite database (`unified_data.db`). 
It maintains tables for:
1. **Instrument Events**: Separate tables for raw `SolexsEvent` and `HeliosEvent` detections.
2. **Unified Flares**: The `UnifiedFlareEvent` entries created by the Event Associator.
3. **Flare Versions**: Tracking the lifecycle mutations of flares to preserve provenance.

## Key Operations
- `save_unified_flare(event)`: Serializes and inserts a new or updated flare event into the database.
- `get_flare_history(start_time, end_time)`: Retrieves a chronologically sorted list of finalized flares within a specific temporal window.
- `get_active_flares()`: Retrieves flares that have not yet reached the `FINALIZED` state (used for resuming state after a system restart).
- `get_flare_provenance(event_id)`: Fetches the specific SoLEXS and HEL1OS events that were merged to create a specific unified flare.

## Integration
The Nowcast Manager periodically flushes the Master Catalog to the Nowcast Repository. The backend REST APIs (`/api/nowcast/catalog`, `/api/nowcast/search`) interface directly with this repository to serve historical data to the frontend widgets (e.g., historical searches in the Catalog Widget).
